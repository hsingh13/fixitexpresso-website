## 6. Content Quality Findings

### common-turbochef-problems.html
- **Best page on the site** — covers F1–F10 with TechArticle + HowTo + FAQPage schema
- **CTR problem:** 15k impressions/90d at pos 5.7 → only 0.56% CTR. Root cause: meta title is generic ("Error Codes F1-F10"), doesn't match specific query intent like "turbochef f7 error code"
- **Improvement needed:** Rewrite meta title to be error-code specific; add jump-nav (F1 | F2 ... F7); expand F7 card with RTD probe explanation and inline CTA

### turbochef-repair.html (hub page)
- **Good content depth** — hero, trust bar, model grid, city links, FAQ schema, dual CTAs
- **Critical issue at launch:** Orphaned — zero internal links, not in sitemap → never crawled
- **Remaining claim:** Footer had "Authorized TurboChef & MerryChef Repair" — fixed

### faq.html
- **12 FAQ items in accordion layout** — zero structured data until this audit
- **H1 is generic:** "Frequently Asked Questions" — should match title tag
- **No LocalBusiness schema** — inconsistent with rest of site

### turbochef-troubleshooting.html
- **Correct implementation:** noindex + meta-refresh redirect to common-turbochef-problems.html
- **Problem:** Listed in sitemap at priority 0.9 — noindex pages should not be in sitemap
- **Fixed:** Removed from sitemap

### City pages (7 pages)
- All indexed but ranking position 29–44 with zero clicks in 90 days
- LocalBusiness + FAQPage schema added June 22 — awaiting Google validation
- Content has city-specific copy and neighborhoods but lacks testimonials and cross-links
- turbochef-repair-seattle.html had duplicate LocalBusiness schema — fixed
- None of the 15 area-grid span tags are hyperlinks (missed cross-linking opportunity)

### llms.txt
- Well-structured AI search optimisation file
- Missing: turbochef-repair.html in Key Pages section
- MerryChef model list inconsistency (missing e6 and conneX in one version)
